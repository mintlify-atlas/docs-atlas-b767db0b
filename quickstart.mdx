title: "Quickstart"
description: "Get your first HTTPower request running in minutes"
---

<Steps>

<Step title="Add the dependency">

Add `httpower` and Finch to your dependencies in `mix.exs`:

```elixir
def deps do
  [
    {:httpower, "~> 0.20.0"},
    {:finch, ">= 0.19.0"}
  ]
end
```

Then fetch your dependencies:

```bash
mix deps.get
```

</Step>

<Step title="Start Finch in your supervision tree">

HTTPower uses Finch for connection pooling. Add `HTTPower.Finch` to your application's supervision tree in `application.ex`:

```elixir
defmodule MyApp.Application do
  use Application

  def start(_type, _args) do
    children = [
      {Finch, name: HTTPower.Finch},
      # ... your other children
    ]

    opts = [strategy: :one_for_one, name: MyApp.Supervisor]
    Supervisor.start_link(children, opts)
  end
end
```

</Step>

<Step title="Make your first request">

Make a GET request by passing a URL string directly:

```elixir
{:ok, response} = HTTPower.get("https://api.example.com/users")

IO.inspect(response.status)  # 200
IO.inspect(response.body)    # %{"users" => [...]}
```

`{:ok, response}` means the server responded — including 4xx and 5xx. Check `response.status` to determine the HTTP outcome.

</Step>

<Step title="Create a configured client">

For repeated calls to the same API, create a pre-configured client with `HTTPower.new/1`:

```elixir
client = HTTPower.new(
  base_url: "https://api.example.com",
  headers: %{"Authorization" => "Bearer #{token}"},
  timeout: 30,
  max_retries: 5
)

# Reuse the client for multiple requests
{:ok, users}   = HTTPower.get(client, "/users")
{:ok, user}    = HTTPower.get(client, "/users/123")
{:ok, created} = HTTPower.post(client, "/users", body: Jason.encode!(params))
```

Request-level options are merged with the client's options. Request options take precedence.

</Step>

<Step title="Handle errors properly">

All HTTPower functions return `{:ok, %HTTPower.Response{}}` or `{:error, %HTTPower.Error{}}` — they never raise for HTTP operations. Pattern match on the result to handle each case:

```elixir
case HTTPower.get("https://api.example.com/users") do
  {:ok, %HTTPower.Response{status: status, body: body}} when status in 200..299 ->
    # Successful response
    process_data(body)

  {:ok, %HTTPower.Response{status: 404}} ->
    # Server responded, resource not found
    {:error, :not_found}

  {:ok, %HTTPower.Response{status: status}} when status in 500..599 ->
    # Server responded with an error (after retries exhausted)
    {:error, :server_error}

  {:error, %HTTPower.Error{reason: :timeout}} ->
    # Request timed out
    {:error, :timeout}

  {:error, %HTTPower.Error{reason: reason}} ->
    # Other transport error (e.g. :econnrefused)
    {:error, reason}
end
```

</Step>

<Step title="Enable production logging">

Call `HTTPower.Logger.attach/0` in your application startup to enable PCI-compliant structured logging for all HTTP requests:

```elixir
defmodule MyApp.Application do
  use Application

  def start(_type, _args) do
    HTTPower.Logger.attach()

    children = [
      {Finch, name: HTTPower.Finch},
      # ...
    ]

    opts = [strategy: :one_for_one, name: MyApp.Supervisor]
    Supervisor.start_link(children, opts)
  end
end
```

Sensitive data — authorization headers, credit card numbers, passwords, and other PCI fields — is automatically redacted. Each request is assigned a unique correlation ID:

```
[HTTPower] [req_a1b2c3d4e5f6g7h8] → GET https://api.example.com/users headers=%{"authorization" => "[REDACTED]"}
[HTTPower] [req_a1b2c3d4e5f6g7h8] ← 200 (45ms) body=%{"users" => [...]}
```

All log entries also include structured `Logger.metadata()` (e.g. `httpower_status`, `httpower_duration_ms`, `httpower_correlation_id`) for querying in Datadog, Splunk, or Loki.

</Step>

<Step title="Write tests">

Enable test mode and use `HTTPower.Test` to stub HTTP calls without making real requests:

```elixir
# test/test_helper.exs
Application.put_env(:httpower, :test_mode, true)
ExUnit.start()
```

```elixir
defmodule MyApp.ApiClientTest do
  use ExUnit.Case

  setup do
    HTTPower.Test.setup()
  end

  test "fetches users successfully" do
    HTTPower.Test.stub(fn conn ->
      HTTPower.Test.json(conn, %{users: [{id: 1, name: "Alice"}]})
    end)

    {:ok, response} = HTTPower.get("https://api.example.com/users")

    assert response.status == 200
    assert response.body["users"] != []
  end

  test "real requests are blocked in test mode" do
    {:error, error} = HTTPower.get("https://api.example.com/users")

    assert error.reason == :network_blocked
  end
end
```

`HTTPower.Test.setup/0` resets stub state between tests. Stubs work across all adapters.

</Step>

</Steps>
