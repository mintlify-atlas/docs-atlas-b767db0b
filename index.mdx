## What is HTTPower?

HTTPower is an Elixir HTTP client library that brings production-reliability patterns — circuit breaker, rate limiting, PCI-compliant logging, smart retries, and request deduplication — to your HTTP calls. It works as a thin layer on top of your existing HTTP client (Finch, Req, or Tesla), adding enterprise-grade resilience without changing how you write HTTP code.

```elixir mix.exs
def deps do
  [
    {:httpower, "~> 0.20.0"},
    {:finch, ">= 0.19.0"}  # or req / tesla
  ]
end
```

```elixir Basic usage
# Simple one-off request
{:ok, response} = HTTPower.get("https://api.example.com/users")

# Configured client for reuse
client = HTTPower.new(
  base_url: "https://api.example.com",
  headers: %{"Authorization" => "Bearer #{token}"},
  max_retries: 3
)

{:ok, users}   = HTTPower.get(client, "/users")
{:ok, created} = HTTPower.post(client, "/users", body: data)
```

## Built for reliability

HTTPower never raises exceptions for HTTP operations. Every call returns `{:ok, response}` or `{:error, reason}`, making error handling explicit and exhaustive.

```elixir Error handling
case HTTPower.get("https://api.payment.com/charge") do
  {:ok, %HTTPower.Response{status: 200, body: body}} ->
    process_payment(body)

  {:ok, %HTTPower.Response{status: 429}} ->
    # Rate limited by server
    schedule_retry()

  {:error, %HTTPower.Error{reason: :service_unavailable}} ->
    # Circuit breaker is open
    use_fallback()

  {:error, %HTTPower.Error{reason: :timeout}} ->
    handle_timeout()
end
```

## Test mode

HTTPower blocks real HTTP requests during tests by default, and provides adapter-agnostic mock helpers:

```elixir ExUnit test
setup do
  HTTPower.Test.setup()
end

test "charges payment" do
  HTTPower.Test.stub(fn conn ->
    HTTPower.Test.json(conn, %{id: "ch_123", status: "succeeded"})
  end)

  {:ok, response} = HTTPower.post("https://api.stripe.com/charges", body: charge_data)
  assert response.body["status"] == "succeeded"
end
```
