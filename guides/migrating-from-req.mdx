---
title: Migrating from Req
description: Add HTTPower's circuit breaker, rate limiting, and PCI-compliant logging to an existing Req application.
---

Since HTTPower uses Req as one of its supported adapters, migration from a Req-based application is straightforward. HTTPower adds circuit breaker, rate limiting, PCI-compliant logging, and enhanced retry logic on top of Req's foundation.

## Why migrate

Req is a great batteries-included HTTP client. HTTPower adds the production reliability layer:

- **Circuit breaker** — stops cascading failures automatically
- **Rate limiting** — token bucket algorithm with per-endpoint configuration
- **Configurable retries** — exponential backoff with jitter; more control than Req's `:transient`
- **PCI-compliant logging** — automatic sanitization of credit cards, tokens, and passwords
- **Telemetry** — structured observability across all requests
- **Never raises** — always returns `{:ok, response}` or `{:error, error}` tuples

## Prerequisites

- Existing Elixir application using Req `~> 0.4.0` or higher

<Steps>

<Step title="Add the HTTPower dependency">

Update `mix.exs` to add HTTPower alongside your existing Req dependency:

```elixir
# mix.exs
def deps do
  [
    {:req, "~> 0.4.0"},         # existing dependency
    {:httpower, "~> 0.5.0"}     # add this
  ]
end
```

Run `mix deps.get`.

</Step>

<Step title="Replace direct Req calls">

The simplest migration replaces `Req` calls with `HTTPower` calls directly.

**Before:**

```elixir
defmodule MyApp.ApiClient do
  def fetch_users do
    Req.get!("https://api.example.com/users")
  end

  def create_user(params) do
    Req.post!("https://api.example.com/users", json: params)
  end
end
```

**After:**

```elixir
defmodule MyApp.ApiClient do
  def fetch_users do
    HTTPower.get("https://api.example.com/users")
  end

  def create_user(params) do
    HTTPower.post("https://api.example.com/users",
      body: Jason.encode!(params),
      headers: %{"content-type" => "application/json"}
    )
  end
end
```

<Note>
HTTPower always returns `{:ok, response}` or `{:error, error}` — it never raises. Replace `Req.get!` with `HTTPower.get` and handle the result tuple.
</Note>

</Step>

<Step title="Migrate Req.new() clients">

If you use `Req.new/1` to build configured clients, map the options to `HTTPower.new/1`:

**Before:**

```elixir
defmodule MyApp.GithubClient do
  def client do
    Req.new(
      base_url: "https://api.github.com",
      headers: [{"user-agent", "MyApp/1.0"}],
      retry: :transient
    )
  end

  def get_repo(owner, repo) do
    Req.get!(client(), url: "/repos/#{owner}/#{repo}")
  end
end
```

**After:**

```elixir
defmodule MyApp.GithubClient do
  def client do
    HTTPower.new(
      base_url: "https://api.github.com",
      headers: %{"user-agent" => "MyApp/1.0"},
      retry: false  # disable Req's retry — HTTPower handles retries
    )
  end

  def get_repo(owner, repo) do
    HTTPower.get(client(), "/repos/#{owner}/#{repo}")
  end
end
```

<Warning>
Set `retry: false` on the client to disable Req's built-in retry. HTTPower manages retries independently. Leaving both active causes double-retrying.
</Warning>

</Step>

<Step title="Add HTTPower reliability features">

Once the basic migration is done, enable HTTPower's production features:

```elixir
defmodule MyApp.GithubClient do
  def client do
    HTTPower.new(
      base_url: "https://api.github.com",
      headers: %{"user-agent" => "MyApp/1.0"},
      retry: false,

      # HTTPower retry with exponential backoff
      max_retries: 3,
      base_delay: 1000,

      # Circuit breaker
      circuit_breaker: [
        failure_threshold: 5,
        timeout: 60_000
      ],

      # Rate limiting (GitHub's limit: 60 req/min unauthenticated)
      rate_limit: [
        requests: 60,
        per: :minute
      ]
    )
  end
end
```

</Step>

<Step title="Configure globally (recommended)">

Set reliability options once in `config/config.exs` instead of on every client:

```elixir
# config/config.exs
config :httpower,
  max_retries: 3,
  base_delay: 1000,
  max_delay: 30000,
  retry_safe: false,

  circuit_breaker: [
    enabled: true,
    failure_threshold: 5,
    window_size: 10,
    timeout: 60_000
  ],

  rate_limit: [
    enabled: true,
    requests: 100,
    per: :minute,
    strategy: :wait
  ],

  logging: [
    enabled: true,
    level: :info
  ]
```

Your clients then only need service-specific options:

```elixir
defmodule MyApp.GithubClient do
  def client do
    HTTPower.new(
      base_url: "https://api.github.com",
      headers: %{"user-agent" => "MyApp/1.0"}
      # Circuit breaker, rate limiting, retries from global config
    )
  end
end
```

</Step>

<Step title="Update tests">

HTTPower maintains compatibility with `Req.Test`. Replace `Req.Test.stub(MyApp, ...)` with `Req.Test.stub(HTTPower, ...)`:

**Before:**

```elixir
test "fetches users" do
  Req.Test.stub(MyApp, fn conn ->
    Req.Test.json(conn, %{"users" => []})
  end)

  client = Req.new(plug: {Req.Test, MyApp})
  assert %Req.Response{status: 200} = Req.get!(client, url: "/users")
end
```

**After:**

```elixir
# test/test_helper.exs
Application.put_env(:httpower, :test_mode, true)

# test/my_app/github_client_test.exs
test "fetches users" do
  Req.Test.stub(HTTPower, fn conn ->
    Req.Test.json(conn, %{"users" => []})
  end)

  assert {:ok, %{status: 200}} = HTTPower.get("/users",
    plug: {Req.Test, HTTPower}
  )
end
```

</Step>

</Steps>

## Key differences

### Error handling

**Req:**
```elixir
# Bang variant raises on error
response = Req.get!("https://api.example.com")

# Non-bang returns result tuple
{:ok, response} = Req.get("https://api.example.com")
```

**HTTPower:**
```elixir
# Always returns a result tuple — never raises
{:ok, response} = HTTPower.get("https://api.example.com")
{:error, %HTTPower.Error{reason: :timeout}} = HTTPower.get("https://slow-api.com")
```

### Retry behavior

| | Req | HTTPower |
|---|---|---|
| Default | 3 retries (`:transient`) | Configurable (default off) |
| Backoff | Exponential | Exponential with jitter |
| Retryable status codes | Transient errors | 408, 429, 500–504 |
| Configuration | `:transient`, `:safe_transient`, or custom fn | `max_retries`, `base_delay`, `max_delay`, `retry_safe` |

### JSON handling

Req automatically encodes and decodes JSON. HTTPower does not — use Jason explicitly:

```elixir
# Encode request body
HTTPower.post(url,
  body: Jason.encode!(params),
  headers: %{"content-type" => "application/json"}
)

# Decode response body
{:ok, response} = HTTPower.get(url)
data = Jason.decode!(response.body)
```

### Req options pass-through

HTTPower internally uses Req. Most `Req.new/1` options pass through:

```elixir
HTTPower.get("https://api.example.com",
  # HTTPower options
  max_retries: 3,
  circuit_breaker: [...],

  # Req options (passed through)
  connect_timeout: 15_000,
  receive_timeout: 30_000,
  decode_body: false
)
```

## Migration patterns

<AccordionGroup>
  <Accordion title="Pattern 1: Gradual module-by-module">
    Migrate one module at a time while others continue using Req:

    ```elixir
    # Still on Req
    defmodule MyApp.InternalClient do
      def fetch_config do
        Req.get!("http://internal-config/settings")
      end
    end

    # Migrated to HTTPower
    defmodule MyApp.ExternalApiClient do
      def fetch_data do
        HTTPower.get("https://external-api.com/data")
      end
    end
    ```
  </Accordion>

  <Accordion title="Pattern 2: Wrapper pattern">
    Keep Req for non-critical calls; wrap critical paths with HTTPower:

    ```elixir
    defmodule MyApp.PaymentClient do
      # Critical — uses HTTPower
      def charge_customer(amount) do
        client = HTTPower.new(
          base_url: "https://api.stripe.com",
          circuit_breaker: [failure_threshold: 3]
        )

        HTTPower.post(client, "/v1/charges", body: encode_params(amount))
      end

      # Non-critical — still on Req
      def list_products do
        Req.get!("https://api.stripe.com/v1/products")
      end
    end
    ```
  </Accordion>

  <Accordion title="Pattern 3: Feature flags">
    Use feature flags for a gradual, reversible rollout:

    ```elixir
    defmodule MyApp.ApiClient do
      def fetch_users do
        if use_httpower?() do
          HTTPower.get("https://api.example.com/users")
        else
          {:ok, Req.get!("https://api.example.com/users")}
        end
      end

      defp use_httpower? do
        Application.get_env(:myapp, :use_httpower, false)
      end
    end
    ```
  </Accordion>
</AccordionGroup>

## Common scenarios

  <Tab title="API client with retries">

    ```elixir
    defmodule MyApp.ApiClient do
      def client do
        HTTPower.new(
          base_url: "https://api.example.com",

          timeout: 30
        )
      end

      def fetch_data do
        HTTPower.get(client(), "/data")
      end
    end
    ```
  </Tab>
  <Tab title="High-volume with rate limiting">
    ```elixir
    config :httpower,
      rate_limit: [
        enabled: true,
        requests: 1000,
        per: :minute,
        strategy: :wait,
        max_wait_time: 5000
      ]

    defmodule MyApp.HighVolumeClient do
      def process_batch(items) do
        Enum.map(items, fn item ->
          HTTPower.post("https://api.example.com/process", body: item)
        end)
      end
    end
    ```
  </Tab>
  <Tab title="Payment processing">
    ```elixir
    config :httpower,
      circuit_breaker: [
        enabled: true,
        failure_threshold: 3,
        timeout: 30_000
      ],
      logging: [
        enabled: true,
        sanitize_body_fields: ["card_number", "cvv"]
      ]

    defmodule MyApp.PaymentClient do
      def charge(amount) do
        case HTTPower.post("https://api.stripe.com/v1/charges", body: amount) do
          {:ok, response} -> {:ok, response}
          {:error, %{reason: :service_unavailable}} -> {:error, :service_unavailable}
          {:error, error} -> {:error, error}
        end
      end
    end
    ```
  </Tab>
</Tabs>

## FAQ

<AccordionGroup>
  <Accordion title="Do I need to change all my Req calls at once?">
    No. HTTPower and Req coexist in the same application. Migrate modules gradually, starting with the ones that call external services where reliability matters most.
  </Accordion>

  <Accordion title="Can I use Req-specific features like plugins?">
    Most Req options pass through to the underlying Req client. Req plugins may not be directly supported — check the documentation or pass them as options to `HTTPower.new/1`.
  </Accordion>

  <Accordion title="Should I disable Req's retry?">
    Yes. Set `retry: false` when creating HTTPower clients. HTTPower's retry logic is more configurable and avoids the double-retry problem.
  </Accordion>

  <Accordion title="Can I use Req.new() options with HTTPower.new()">
    Yes. HTTPower internally uses Req, so options are passed through. Use `HTTPower.new/1` instead of `Req.new/1` to get HTTPower's reliability layer on top.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Double retrying">
    Symptom: requests are retried more times than expected.

    Disable Req's built-in retry:

    ```elixir
    HTTPower.new(
      base_url: "https://api.example.com",
      retry: false  # disable Req's retry
    )
    ```
  </Accordion>

  <Accordion title="Test mode not blocking real requests">
    Enable test mode in `test_helper.exs` before any tests run:

    ```elixir
    # test/test_helper.exs
    Application.put_env(:httpower, :test_mode, true)
    ```
  </Accordion>

  <Accordion title="JSON not automatically decoded">
    HTTPower does not automatically encode or decode JSON like Req does. Use Jason explicitly:

    ```elixir
    # Encode request body
    HTTPower.post(url,
      body: Jason.encode!(params),
      headers: %{"content-type" => "application/json"}
    )

    # Decode response body
    {:ok, response} = HTTPower.get(url)
    data = Jason.decode!(response.body)
    ```
  </Accordion>
</AccordionGroup>
